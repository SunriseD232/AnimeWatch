Работа с плагином

  • Перейти по ссылке https://plugins.vibix.org/v1/vibix.zip
  • Выбрать путь для сохранения плагина

  1. Открыть панель управления DLE
  2. Перейти в пункт "Утилиты"
  3. Выбрать пункт "Управление плагинами"
  4. Внизу страницы выбрать "Загрузить плагин"
  5. Нажать кнопку "Выберите файл"
  6. В открывшемся окне указать путь к сохраненному плагину
  7. Нажать кнопку "Загрузить плагин"
  8. Выбрать на панели пункт "Сторонние модули"
  9. Выбрать пункт "Vibix - видеобалансер"
  10. В появившемся окне ввести свой API token из личного кабинета
  11. Проставить базовое сопоставление полей

Плагин постоянно развиваются и расширяет функционал. Соответственно, при работе с плагином, вы можете увидеть уведомление "У вас установлена устаревшая версия плагина"

В этом случае надо:

  1. Перейти в пункт "Утилиты -> Управление плагинами"
  2. Найти в списке плагин Vibix
  3. Справа в выпадающем меню выбрать "Проверить наличие новых версий"
  4. В появившемся окне подтвердить обновление до новой версии

Функционал плагина

Плагин предназначен для быстрого добавления новостей на сайт, проставления ссылок и отслеживания статистики


  • В каталоге представлен список полный список фильмов
  • Над списком фильмов расположены разнообразные фильтры, которые существенно облегчают поиск
  • Фильмы, которые уже есть на сайте, будут помечены зеленым флажком рядом с названием
  • Если фильма нет в наличии, его можно запросить, нажав на соответствующую кнопку
  • Внизу страницы расположены основные элементы управлением создания заданий

  • В разделе указывается личный API token и автор публикаций
  • Представлена возможность настройки шаблонов SEO

  • В разделе настраивается сопоставление ваших дополнительных полей с полями по базе Vibix
  • Обязательными для сопоставлениями полями являются "ссылка на плеер Vibix" и "Кинопоиск ID"

  • В разделе настраивается сопоставление ваших категорий с полями по базе Vibix

  • В разделе отображается ваша статистика в выбранном диапазоне дат

  • В профиле отображаются ваши данные

  • В разделе отображены активные заявки на добавление фильмов

  • В разделе отображены подготовленные к выполнению задания, созданные в Каталоге.
  • Задания можно запустить вручную, либо настроить автоматический запуск

Выбор фильмов

Выборка для создания заданий осуществляется двумя способами:




В Каталоге отметьте нужные фильмы флажками



Установите селект внизу страницы в положение "Только с выбранными"




Создайте необходимую комбинацию фильтров. В этом случае расставлять флажки не нужно. При использовании этого способа настроятельно рекомендуется использовать фильтр "Только в наличии". Это существенно ускорит последующее выполнение задания.



Установите селект внизу страницы в положение "Со всей выборкой по фильтру"

Добавление новостей

Если вы хотите добавить на сайт фильмы, которых нет у вас в базе:


1) выберите нужные фильмы, отмечая их вручную или с помощью фильтра (см "Выбор фильмов")


Не забывайте установить селект внизу страницы в положение, соответствующее вашей выборке!

2) Нажмите кнопку "Добавить если нет"



Существующие записи при этом затронуты не будут! Добавятся только новости с фильмами, которых еще нет на сайте.


Обратите внимание, что у вас уже должно быть настроено сопоставление полей (и категорий).


Постер предоставляется в виде ссылки. Сам постер на сайте не сохраняется. Тип поля в DLE - "одна строка".


3) Через короткий промежуток времени появится уведомление об успешном создании задания



4) Переходите к пункту "Запуск задания"


Добавление новостей с обновлением существующих

Если вы хотите добавить на сайт фильмы, которых нет у вас в базе и обновить уже существующие в базе записи:


1) выберите нужные фильмы, отмечая их вручную или с помощью фильтра (см "Выбор фильмов")


Не забывайте установить селект внизу страницы в положение, соответствующее вашей выборке!

2) Нажмите кнопку "Добавить и обновить если есть"



Существующие записи при этом будут изменены на данные из каталога "Vibix"


Обратите внимание, что у вас уже должно быть настроено сопоставление полей (и категорий).


Постер представлен в виде ссылки. Тип поля в DLE - "одна строка". Постер можно сохранять на сервере.


3) Через короткий промежуток времени появится уведомление об успешном создании задания



4) Переходите к пункту "Запуск задания"


Добавление ссылок

Опция используется только если вы хотите проставить в уже существующие новости ссылки на плеер "Vibix":


1) выберите нужные фильмы, отмечая их вручную или с помощью фильтра (см "Выбор фильмов")


Не забывайте установить селект внизу страницы в положение, соответствующее вашей выборке!

2) Нажмите кнопку "Проставить ссылку на плеер "Vibix"



Данные в новостях при этом изменены не будут! Ссылка будет прописана в созданное вами доп. поле DLE




Если есть необходимость добавить ссылку на плеер сразу ко всем новостям на сайте:


1) выберите фильтр "Только в наличии"



2) Установите селект внизу страницы в положение "Со всей выборкой по фильтру" и выберите опцию - Проставить ссылку на плеер "Vibix"



3) Через короткий промежуток времени появится уведомление об успешном создании задания



4) Переходите к пункту "Запуск задания"


Запуск заданий

Созданные задания появятся в разделе "Фоновые задания". Их запуск можно осуществить двумя способами:


Просмотрите список созданных заданий. При необходимости удалите ненужные задания


Нажмите кнопку "Запустить задания вручную". Ход выполнения задания отобразиться на шкале прогресса



Для работы фоновых заданий в автоматическом режиме настройте планировщик заданий Cron на исполнение скрипта /cron_vibix.php, он находится в корневой директории вашего сайта. Возможно вам поможет инструкция REG RU Планировщик CRON.


Если возникли проблемы с настройкой Cron, рекомендуем написать тикет в тех поддержку вашего хостинга с текстом: Просьба помочь настроить планировщик крон на ежеминутное исполнение для файла /cron_vibix.php


Описание

Что это: Структурированные данные для поисковых систем в формате JSON-LD.

Для чего нужно: Помогает поисковым системам лучше понимать содержание страницы, улучшает отображение в результатах поиска (расширенные сниппеты, рейтинги, даты выхода и т.д.).


Что это: Мета-теги для социальных сетей и мессенджеров.

Для чего нужно: Контролирует отображение контента при расшаривании в социальных сетях (Facebook, VK, Telegram и др.) - заголовок, описание, изображение.


Настройка

Если хотите использовать какой-то из видов микроразметки на сайте, добавьте соответствующее доп. поле (тип - одна строка) и пропишите в шаблон


Для работы микроразметки необходимо создать дополнительные поля:


vibix_schema_microdata - для хранения JSON-LD разметки

vibix_og_microdata - для хранения Open Graph тегов


Вставьте следующий код в файл main.tpl после открывающего тега <head>:


[xfgiven_vibix_schema_microdata]
<script type="application/ld+json">
[xfvalue_vibix_schema_microdata]
</script>
[/xfgiven_vibix_schema_microdata]

Вставьте следующий код после открывающего тега <head>:


[xfgiven_vibix_og_microdata]
[xfvalue_vibix_og_microdata]
[/xfgiven_vibix_og_microdata]

Установка кода

Для установки кода вам потребуется доступ к HTML разметке вашего сайта.

Найдите в коде закрывающий тег </head> и вставьте перед ним следующий код:


<script type="text/javascript" src="https://v-js-menu.run/public/lib.en.min.js"></script>

Внутри тега <body> разместите код подключения

<ins id="vibix_union" data-publisher_id="678512960" data-add_types="brand,sticker,pcsticker,banners,flyroll"></ins>


В параметр data-add_types через запятую указываем форматы, которые вы готовы размещать у себя на странице.


Доступны следующие значения:


  • sticker - Анимированный креатив в нижней части сайта на мобильном устройстве (Catfish).

  • pcsticker - Анимированный креатив в нижней части сайта на ПК.

  • banners - Баннеры определенных размеров, установленные по сайту.

  • brand - Статичный креатив в виде фона сайта, областями по краям сайта и в шапке. Данный формат пытается адаптировать сайт под размеры креативов на заднем фоне! Если сайт после запуска выглядит не корректно - отключите данный формат и обратитесь к нам в поддержку.

  • flyroll - Автоматически вставляемый рекламный ролик


Баннеры пытаются адаптироваться под блок, старайтесь подбирать баннеры максимально подходящие по размеру. Доступные размеры и коды для вставки:


  • - 300x250 <ins data-pm-b="300x250"></ins>
  • - 300x600 <ins data-pm-b="300x600"></ins>
  • - 680x200 <ins data-pm-b="680x200"></ins>
  • - 680x250 <ins data-pm-b="680x250"></ins>
  • - 728x90   <ins data-pm-b="728x90"></ins>


Fly-Roll будет фиксироваться в правом нижнем углу экрана (или в нижней части экрана на мобильных устройствах). Дополнительно можно закрепить его сверху или снизу страницы, добавив элемент в месте где хотелось бы его поставить:


<ins data-pm-flyroll=""></ins>


Если этот элемент выходит из зоны видимости, Fly-Roll автоматически закрепится в правом нижнем углу.
Выбрать положение можно добавив в основной скрипт data-position="*", где вместо * нужно указать номер позиции:

  • 1 - Право сверху
  • 2 - Право снизу
  • 3 - Лево снизу
  • 4 - Лево сверху

Базовое использование

<ins data-publisher-id="678512960"
     data-type="ТИП_КОНТЕНТА"
     data-id="ID_КОНТЕНТА"></ins>

Обязательные параметры

  • data-publisher-id - Ваш Publisher ID. Допустимые значения: число. Пример: 678512960

  • data-type - Тип контента. Допустимые значения: movie, series, kp, imdb. Пример: "series"

  • data-id - ID видео/сериала. Допустимые значения: число. Пример: "1"


<ins data-publisher-id="678512960"
     data-type="movie"
     data-id="1"></ins>
<ins data-publisher-id="678512960"
     data-type="series"
     data-id="12345"></ins>

Опциональные параметры

  • data-season - Номер сезона. Применимо к: series. Пример: "2"

  • data-episodes - Список серий через запятую. Применимо к: series. Пример: "1,5,10"

  • data-voiceover - ID озвучки. Применимо к: все типы. Пример: "4"

  • data-voiceover-only - Только указанная озвучка (без возможности выбора). Применимо к: все типы. Пример: "true"


  • data-design - Дизайн плеера. Значения: 1-6. Пример: "1"

  • data-poster - Показать постер перед воспроизведением. Значения: "true" для показа постера из базы Vibix ИЛИ ссылка на свой постер. Пример: "true" ИЛИ "https://images.steamusercontent.com/ugc/543024637576359294/E4C9D321AC20C7356C5D2F9461DAACE1BF4ED7FC/"

  • data-nopreload - Легковесная интеграция (загрузка по клику). Значения: "true". Пример: "true"

  • data-sync - Совместный просмотр. Значения: "true". Пример: "true"

  • data-autoplay - Автовоспроизведение. Значения: "true". Пример: "true"

  • data-color{1-5} - Цвета плеера. Значения: HEX-код. Пример: data-color1="#56CEAA"


  • data-width - Ширина iframe. По умолчанию: "100%". Пример: "800px"

  • data-height - Высота iframe. По умолчанию: "370px". Пример: "450px"

Выбор озвучки по умолчанию

<ins data-publisher-id="678512960"
     data-type="movie"
     data-id="187471"
     data-voiceover="4"></ins>

Озвучка с ID 4 будет в начале списка, но пользователь может выбрать другую.


Только одна озвучка (без выбора)

<ins data-publisher-id="678512960"
     data-type="movie"
     data-id="187471"
     data-voiceover="4"
     data-voiceover-only="true"></ins>

Будет доступна только озвучка с ID 4.


Используйте data-nopreload="true" для отложенной загрузки плеера:

<ins data-publisher-id="678512960"
     data-type="movie"
     data-id="187471"
     data-nopreload="true"
     data-poster="true"></ins>

Как работает: Сначала загружается только изображение-превью. Плеер загружается после клика пользователя. Это ускоряет загрузку страницы.


<ins data-publisher-id="678512960"
     data-type="movie"
     data-id="187471"
     data-autoplay="true"></ins>

⚠️ Внимание: Большинство браузеров блокируют автовоспроизведение со звуком.

Совместный просмотр видео

Организуйте синхронный просмотр видео с друзьями и зрителями в реальном времени.

Быстрый старт (за 3 шага)

Добавьте в тег подключения плеера параметр data-sync="true"


Добавьте эту строку перед закрывающим тегом /body


<script src="https://sync.videoframe2.com/sync-lib.js"></script>

Чтобы создать комнату, добавьте код перед закрывающим тегом /body


<script>
    document.addEventListener('DOMContentLoaded', function() {
        const party = new WatchParty({
        iframe: '#vibix-frame-id',     // Селектор iframe
        roomId: 'example_room_12345',  // ID комнаты (уникальный идентификатор команты, если не укаазан, будет сгенерирован автоматически)
        username: 'Иван Петров',       // Имя зрителя (необязательно)
        debug: false                   // Логи в консоль (для отладки)
        });
    });
</script>

Теперь на вашей странице работает совместный просмотр.



Как это работает?

  1. Откройте страницу с видео
  2. Автоматически создастся комната с уникальным ID
  3. Нажмите кнопку "Скопировать ссылку" в виджете справа
  4. Отправьте ссылку друзьям (через мессенджер, соцсети и т.д.)
  5. Управляйте просмотром — ваши действия синхронизируются со всеми зрителями
  1. Перейдите по ссылке, которую отправил ведущий
  2. Автоматически подключитесь к комнате
  3. Смотрите синхронно — все действия ведущего повторяются у вас

Синхронизируется:

  • Воспроизведение — если кто-то нажал плей, видео запускается у всех
  • Пауза — если кто-то нажал пауза, видео останавливается у всех
  • Перемотка — если кто-то перематывает, у всех перематывается на ту же позицию
  • Смена серии — если кто-то переключил серию, у всех переключается на ту же серию
  • Автосинхронизация — если у кого-то возникло отставание, происходит автоматическое выравнивание

Параметр data-trailer (трейлер)

Управляет показом трейлера. Поведение зависит от типа контента.


Для типов kp и imdb

"true" — Трейлер будет показан только если видео отсутствует в каталоге. Если по ID Кинопоиска или IMDB найден соответствующий фильм или сериал (уже есть в базе), загрузится полный плеер, а трейлер показан не будет.

"only" — Трейлер будет показан всегда, даже если полная версия доступна в каталоге.


Для типов movie и series

Параметр применим, но только со значением "only". Поскольку ID напрямую указывает на существующий в каталоге контент, трейлер никогда не требуется как заглушка. Значение "true" для этих типов игнорируется (плеер покажет полное видео как обычно).

Примеры

<ins data-publisher-id="678512960"
                 data-type="kp"
                 data-id="326"
                 data-trailer="true"></ins>

Если фильма с таким ID Кинопоиска нет в каталоге, зритель увидит трейлер. Если фильм найден — зритель увидит полноценный фильм или сериал.


<ins data-publisher-id="678512960"
                 data-type="imdb"
                 data-id="tt0111161"
                 data-trailer="only"></ins>

Трейлер будет показан при любых условиях, полная версия недоступна.


<ins data-publisher-id="678512960"
                 data-type="movie"
                 data-id="187471"
                 data-trailer="only"></ins>

Несмотря на наличие полного фильма, плеер загрузит только трейлер.


<ins data-publisher-id="678512960"
                 data-type="series"
                 data-id="12345"
                 data-trailer="only"
                 data-poster="true"></ins>

Показывается трейлер сериала, возможность посмотреть серии отсутствует.


<ins data-publisher-id="678512960"
                 data-type="movie"
                 data-id="187471"
                 data-trailer="true"></ins>

Параметр data-trailer="true" не оказывает влияния на типы movie и series. Плеер проигнорирует его и покажет полный фильм.
